# 《术士之战》待 Unity 在线后联测清单

## 1. 文档定位

这份清单给 Unity 在线后的运行态联测使用，重点是把当前最需要闭环的战斗链路拆成可直接执行的步骤。

它不是正式日志，不写入结论归档。  
正式记录仍由主控统一写入 `D:\术士之战\magic war\ai文件\操作记录.md`。

---

## 2. 本轮联测重点

本轮优先覆盖以下链路：

1. 暂停层
2. 四职业选人
3. 法师生成
4. `BattleFlow` 对位关系
5. 点击移动 / 瞄准 / 近战
6. 岩浆 / 燃烧状态

若 Unity 在线时间有限，建议至少先跑完第 4 到第 9 节。

---

## 3. 联测前置条件

开始前先确认：

1. Unity 已打开项目 `D:\术士之战\magic war`
2. 当前场景可进入 `Assets/Scenes/SampleScene.unity`
3. Console 无新增编译错误
4. 当前线程若要走 MCP，已确认能访问 Unity 工具
5. 本轮只做验证，不改正式逻辑代码

建议先留 1 次基础现场记录：

- 当前场景名
- Play 前 Console 状态
- 若有截图，统一放在 `Assets/Screenshots/`

## 6. 暂停层联测

### 6.1 目标

确认 `Escape` 暂停链路、暂停层中文文案、继续战斗、返回选人都可用，并且暂停时战斗输入和敌方行为被正确冻结。

### 6.2 操作步骤

1. 进入一局战斗，等到 `战斗中`
2. 按 `Esc`
3. 观察是否出现暂停层
4. 观察标题、副标题、按钮是否为中文
5. 暂停期间尝试：
   - 玩家移动
   - 玩家施法
   - 观察敌人是否继续追击
6. 点击 `继续战斗`
7. 再次按 `Esc`
8. 点击 `返回选人`

### 6.3 预期结果

1. 暂停层出现，标题为 `已暂停`
2. 副标题为 `当前对局已暂停`
3. 两个按钮分别为：
   - `继续战斗`
   - `返回选人`
4. 暂停后玩家与敌人都冻结，不继续移动或施法
5. 点击 `继续战斗` 后恢复战斗
6. 点击 `返回选人` 后关闭对局并回到选人层
7. 返回选人后，不残留上一局敌人或结果层

### 6.4 异常观察点

重点看：

1. 按 `Esc` 时若玩家正处于点击瞄准态，是否先退出瞄准而不是直接开暂停
2. 暂停后 HUD 是否仍在异常刷新
3. 返回选人后 `Time.timeScale` 是否恢复，避免下次开局仍停在暂停态

### 6.5 代码锚点

- `Assets/script/core/GameManager.cs`
  - `Update()`
  - `SetPaused(bool paused)`
  - `ApplyPauseState(bool paused)`
  - `ResumeFromPause()`
  - `ReturnToSelectionFromPause()`
  - `EnsurePauseOverlay()`

## 7. 四职业选人与法师生成联测

### 7.1 目标

确认选人界面可见、四职业都能点击、点击后只生成对应职业，且法师生成链路正常。

### 7.2 操作步骤

建议按职业逐个验证：

1. 打开选人层
2. 依次点击：
   - 法师
   - 术士
   - 代行者
   - 邪教徒
3. 每次点击后都确认：
   - 选人界面关闭
   - 场景中只生成 1 个玩家角色
   - HUD 英雄名与当前选择一致
4. 再返回选人界面，重复下一个职业

### 7.3 法师专项确认

法师需要单独补确认：

1. 点击法师卡后，生成对象应来自 `MagePlayer.prefab`
2. HUD 英雄名应显示法师中文名
3. 不应回退成术士或其他职业
4. 角色能正常进入倒计时，再进入 `战斗中`

### 7.4 预期结果

1. 四张职业卡都能点击
2. 任何时刻场景中只保留当前玩家 1 个实例
3. 每次切换职业后，上一个玩家实例被清理
4. 法师可正常生成、绑定 HUD、进入战斗

### 7.5 建议截图

至少留：

1. 选人界面全图 1 张
2. 法师生成后 HUD 图 1 张
3. 至少另一个职业生成后 HUD 图 1 张

### 7.6 代码锚点

- `Assets/script/core/CharacterSelectionManager.cs`
  - `CreateSelectionUi()`
  - `SelectMage()`
  - `SelectWarlock()`
  - `SelectCultist()`
  - `SelectAgent()`
  - `SpawnCharacter(CharacterClassType characterClass)`
  - `DestroyCurrentPlayer(bool destroyAllScenePlayers)`

- 关联资源：
  - `Assets/Prefabs/Characters/MagePlayer.prefab`
  - `Assets/Prefabs/Characters/WarlockPlayer.prefab`

## 8. BattleFlow 对位联测

当前配置来自 `Assets/Resources/BattleFlowConfig.json`：

1. `Mage -> Warlock`
2. `Warlock -> Agent`
3. `Agent -> Cultist`
4. `Cultist -> Mage`

### 8.2 操作步骤

1. 选择一个职业
2. 进入战斗前后观察敌方生成对象 / 结果层敌方标签 / HUD 相关表现

## 9. 倒计时、锁定与进入战斗联测

### 9.1 目标

确认 `Idle -> Countdown -> Active -> Resolved` 这条链路在运行时可见，并且倒计时阶段玩家 / 敌人都被锁定，进入 `战斗中` 后恢复移动。

### 9.2 操作步骤

1. 选一个职业进入战斗
2. 观察 HUD 阶段文本是否先显示 `战斗开始 + 数字`
3. 倒计时期间尝试玩家移动 / 点击移动 / 施法
4. 观察敌人是否静止
5. 倒计时结束后观察是否自动切成 `战斗中`
6. 切入 `战斗中` 后立刻尝试移动、施法，观察敌人是否开始行动

### 9.3 预期结果

1. 倒计时阶段不能提前开打
2. 倒计时结束后自动进入 `战斗中`
3. 不需要额外按钮或额外点击触发 `Active`
4. `战斗中` 后双方恢复移动和战斗

## 10. 点击移动 / 瞄准 / 近战联测

### 10.1 目标

确认点击模式下的移动、瞄准、确认施法、取消瞄准、近战命中链路都能用。

### 10.2 推荐跑法

建议只选 1 个近战手感明显的职业先跑一轮，再用法师补 1 轮点击瞄准确认。

### 10.3 点击移动

操作：

1. 进入战斗
2. 按 `Tab` 切到点击移动模式
3. 右键地面
4. 观察角色是否向目标点移动
5. 连续点 2 到 3 个位置，观察目标更新

预期：

1. HUD 控制模式切到点击移动对应中文
2. 右键后角色朝目标移动
3. 行进中朝目标方向转向

### 10.4 点击瞄准 - 远程

操作：

1. 点击移动模式下按 `Q`
2. 观察地面是否出现瞄准指示器
3. 移动鼠标，观察圆环和连线是否跟随
4. 左键确认远程施法
5. 再次按 `Q` 或右键取消瞄准做 1 次确认

预期：

1. HUD 动作模式切到远程瞄准对应中文
2. 瞄准指示器显示
3. 左键确认后释放远程技能
4. 成功施法后自动退出瞄准态
5. 右键可取消瞄准

### 10.5 点击瞄准 - 近战

操作：

1. 点击移动模式下按 `E`
2. 观察瞄准圈半径是否切到近战样式
3. 将准星指向敌人前方
4. 左键确认近战
5. 命中后观察敌人受击、击退或掉血

预期：

1. HUD 动作模式切到近战瞄准对应中文
2. 左键确认后触发 `TryCastMelee`
3. 命中时敌人掉血
4. 未命中时只进入 miss recovery，不应直接报错

### 10.6 额外观察点

重点看：

1. 角色处于不能施法状态时，瞄准圈是否会自动隐藏
2. 暂停 / 死亡 / 恢复期时是否还能错误地进入瞄准态
3. 点击模式下近战和远程的提示文案是否已切成中文

### 10.7 代码锚点

- `Assets/script/charcter/PlayerController3D.cs`
  - `TryToggleControlMode()`
  - `HandleClickMoveMode()`
  - `TryEnterPrimaryAim()`
  - `TryEnterMeleeAim()`
  - `TryConfirmAimCast(Vector3 aimDirection)`
  - `TryCancelAimMode()`

- `Assets/script/UI/BattleAimIndicator.cs`
  - `Update()`

## 11. 岩浆 / 燃烧状态联测

### 11.1 目标

确认角色接触外圈岩浆后：

1. 立即受岩浆伤害
2. HUD 状态能反映 `熔浆 / 燃烧 / 熔浆 + 燃烧`
3. 离开岩浆后若仍处于燃烧持续时间，状态能继续表现

### 11.2 操作步骤

1. 开局后主动将玩家移动到外圈岩浆
2. 在接触瞬间观察：
   - 血量是否下降
   - HUD 状态是否变化
3. 在岩浆中停留一小段时间，观察是否持续 tick 掉血
4. 离开岩浆
5. 继续观察 2 到 3 秒，确认燃烧是否继续生效
6. 若条件允许，也观察敌人进入岩浆时是否同样掉血

### 11.3 预期结果

1. 接触岩浆时触发直接伤害
2. 若岩浆配置启用燃烧，则同时附加 burn
3. 接触中 HUD 不应在 `熔浆` 和 `燃烧` 之间乱闪
4. 离开岩浆后，若 burn 仍未结束，可继续掉血
5. burn 结束后状态恢复正常

### 11.4 关键判定点

重点看：

1. 血量条是否按 tick 连续减少
2. 状态文案是否出现：
   - `熔浆`
   - `燃烧`
   - `熔浆 + 燃烧`
3. 状态切换是否平滑，不抖动

### 11.5 建议截图

1. 接触岩浆瞬间 1 张
2. 岩浆中持续受伤 1 张
3. 离开岩浆但仍燃烧 1 张

## 13. 建议优先记录的异常分类

如果联测出问题，建议先按下面分类记，主控更容易分发：

1. 选人链路异常
   - 点了没反应
   - 生成错职业
   - 多实例残留

2. BattleFlow 异常
   - 对位错误
   - 倒计时不切战斗中
   - 战斗结束不结算

3. 输入与操作异常
   - 点击移动无效
   - 瞄准圈不显示
   - 近战不命中

4. 状态与环境异常
   - 岩浆不掉血
   - 燃烧不续伤
   - HUD 状态文案乱跳

5. UI / 暂停层异常
   - 文案不对
   - 按钮无效
   - 返回选人后现场没清干净

---

## 14. 给主控回收时最有价值的素材

优先回收这些内容：

1. 四职业 -> 敌方对位对照表
2. 法师生成是否独立正常
